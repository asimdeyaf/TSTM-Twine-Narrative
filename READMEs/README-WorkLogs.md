# Work Logs Guide

## Location
Store logs in:
- `/Users/asimdeyaf/Documents/Twine/READMEs/WorkLogs/`

## Ownership
- Primary operator name to include in logs: `Asim Deyaf`

## Naming
Use one file per day:
- `Asim_WorkLog_YYYY-MM-DD.md`

Example:
- `Asim_WorkLog_2026-02-28.md`

## Minimum Entry Content
For each meaningful work block, include:
- timestamp (local time)
- operator name (`Asim Deyaf`; add collaborator if applicable)
- task summary
- files changed
- git sync/check status
- commit hash (if committed)

## Recommended Flow
1. Start session with Git sync check.
2. Edit story content.
3. Add a log entry immediately after edits.
4. Commit/push and update the same log entry with commit hash.
